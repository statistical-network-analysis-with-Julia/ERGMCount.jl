Terms API Reference

This page documents all count-specific ERGM terms available in ERGMCount.jl.

Basic Terms

SumTerm

ERGMCount.SumTerm — Type
SumTerm <: AbstractERGMTerm

Sum of edge values: ∑{i,j} y{ij} This is the natural sufficient statistic for the Poisson reference.

source

NonzeroTerm

ERGMCount.NonzeroTerm — Type
NonzeroTerm <: AbstractERGMTerm

Number of non-zero edges: ∑{i,j} I(y{ij} > 0)

source

GreaterthannTerm

ERGMCount.GreaterthannTerm — Type
GreaterthannTerm <: AbstractERGMTerm

Number of edges with value > n: ∑{i,j} I(y{ij} > n)

Fields

  • threshold::Int: Threshold value n
source

CountAtleastnTerm

ERGMCount.CountAtleastnTerm — Type
CountAtleastnTerm <: AbstractERGMTerm

Number of edges with value >= n.

source

Structural Terms

CountMutualTerm

ERGMCount.CountMutualTerm — Type
CountMutualTerm <: AbstractERGMTerm

Sum of minimum edge values in reciprocal pairs: ∑{i<j} min(y{ij}, y_{ji})

source

TransitiveTiesTerm

ERGMCount.TransitiveTiesTerm — Type
TransitiveTiesTerm <: AbstractERGMTerm

Triadic-minimum transitivity over ordered distinct triples: ∑{i≠j≠k} min(y{ij}, y{jk}, y{ik})

Note this is a simple valued transitivity; it is related to but not identical to ergm.count's transitiveweights(min, max, min) statistic.

source

CyclicalTiesTerm

ERGMCount.CyclicalTiesTerm — Type
CyclicalTiesTerm <: AbstractERGMTerm

Weighted cyclicality: (1/3) ∑{i≠j≠k} min(y{ij}, y{jk}, y{ki}) (each 3-cycle counted once).

source

Degree / Strength Terms

NodeOSumTerm

ERGMCount.NodeOSumTerm — Type
NodeOSumTerm <: AbstractERGMTerm

Sum of squared out-strengths: measures activity heterogeneity. Directed networks only (0 for undirected; use NodeSumTerm there).

source

NodeISumTerm

ERGMCount.NodeISumTerm — Type
NodeISumTerm <: AbstractERGMTerm

Sum of squared in-strengths: measures popularity heterogeneity. Directed networks only (0 for undirected; use NodeSumTerm there).

source

NodeSumTerm

ERGMCount.NodeSumTerm — Type
NodeSumTerm <: AbstractERGMTerm

Sum of squared total (in + out) strengths.

Interface Functions

Count terms implement the shared ERGM.jl term interface (compute, name) plus the count-specific change statistic change_stat_count, which replaces the binary change_stat for dyads that move between arbitrary count values.

compute

compute(term::AbstractERGMTerm, net) -> Float64

Compute the full-network value of the term statistic. This is the shared ERGM.jl term interface (its docstring lives in the ERGM.jl manual); every count term in this package implements a method that reads the :weight edge attribute (edges without the attribute count as 1).

changestatcount

ERGMCount.change_stat_count — Function
change_stat_count(term, net, weights, i, j, old, new) -> Float64

Change in compute(term, net) when dyad (i,j) moves from count old to count new, holding all other dyads fixed. weights is the :weight edge-attribute dictionary; hot loops should pass the typed snapshot get_edge_attribute(net, :weight, Int) (a Dict{Tuple{T,T},Int}) rather than the untyped get_edge_attribute(net, :weight) Dict.

source

name

name(term::AbstractERGMTerm) -> String

Return the descriptive name of the term (e.g. "sum", "mutual.count"), used to label coefficients in fitted results. This is the shared ERGM.jl term interface (its docstring lives in the ERGM.jl manual); every count term implements a method.

dyad_value

ERGMCount.dyad_value — Function
dyad_value(net, weights, i, j) -> Int

The count value of dyad (i,j): 0 when the edge is absent, its :weight attribute when present (edges without the attribute count as 1).
